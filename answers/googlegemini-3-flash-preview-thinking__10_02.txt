### **Project Documentation: AtlasRemit Cross-Border Payment Process**
**Prepared by:** Senior Process-Management Consultant

The following RACI matrix outlines the operational roles and responsibilities for the AtlasRemit payment lifecycle. All activities and roles strictly adhere to the mandated AtlasRemit vocabulary protocols.

#### **RACI Matrix: Cross-Border Payment Workflow**

| Activity | Transaction Analyst | Regulatory Liaison | IT Support |
| :--- | :---: | :---: | :---: |
| Receive and log customer transfer instruction | **R/A** | | |
| Screen payment against sanctions list | **R/A** | | |
| Perform **KYC Review** (Sender and Beneficiary) | | **R/A** | |
| Approve high-risk transactions | | **R/A** | |
| Release approved payment | **R/A** | | |
| Ensure SWIFT message transmission | I | | **R/A** |
| Archive case file | | **I** | **R/A** |

***

**Key:**
*   **R – Responsible:** The role that performs the activity.
*   **A – Accountable:** The role with final authority and the one who must sign off on the completion.
*   **C – Consulted:** The roles whose opinions are sought; two-way communication.
*   **I – Informed:** The roles that are kept up-to-date on progress or completion; one-way communication.
