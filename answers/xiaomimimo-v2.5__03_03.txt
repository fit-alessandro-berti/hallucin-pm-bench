### Key structural decisions

| Element | Type | Purpose |
|---|---|---|
| `wrc_completenessCheck` | Exclusive Gateway | Loops back to validate until information is complete |
| `wrc_warrantyDecision` | Exclusive Gateway | Routes to rejection or parallel execution |
| `wrc_parallelSplit` / `wrc_parallelJoin` | Parallel Gateways | Fork and join the two concurrent activities |
| `wrc_endRejection` | End Event | Terminates out-of-warranty claims |
| `wrc_endSuccess` | End Event | Terminates successful replacements |

The loopback for incomplete claims (flow 03 → 04) re-enters **Validate the claim**, exactly as specified. No escalation paths, manager approvals, or error-handling paths have been added beyond the textual requirements.
