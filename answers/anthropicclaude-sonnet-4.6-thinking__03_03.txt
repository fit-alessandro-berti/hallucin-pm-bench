### Model summary

| # | Element | Type | Notes |
|---|---------|------|-------|
| 1 | `wrc_start` | Start Event | Customer submits online warranty claim |
| 2 | `wrc_task_validate` | Task | **validates the claim** – has two incoming flows (start + loop-back) |
| 3 | `wrc_gw_complete` | Exclusive GW | *Complete* → step 3; *Incomplete – missing info* → loop |
| 4 | `wrc_task_request` | Task | **requests missing information** → loops back to validate |
| 5 | `wrc_task_check` | Task | **checks warranty eligibility** |
| 6 | `wrc_gw_warranty` | Exclusive GW | *Within warranty* → parallel split; *Out of warranty* → rejection path |
| 7 | `wrc_task_reject` | Task | **send rejection notice** |
| 8 | `wrc_end_rejection` | End Event | ① Rejection end (out of warranty) |
| 9 | `wrc_gw_split` | Parallel GW (+) | Splits into two concurrent branches |
| 10 | `wrc_task_shipment` | Task | **Create shipment order** |
| 11 | `wrc_task_courier` | Task | **Schedule courier pickup** |
| 12 | `wrc_gw_join` | Parallel GW (+) | Synchronises both branches |
| 13 | `wrc_task_inventory` | Task | **update inventory records** |
| 14 | `wrc_task_close` | Task | **close the claim** |
| 15 | `wrc_end_success` | End Event | ② Successful-completion end |
