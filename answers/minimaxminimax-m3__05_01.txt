**Suggestion: Attach a timer boundary event to `Task_RequestDocs`.**

Add an interrupting `bpmn:boundaryEvent` (e.g., id `ESC_Timer`) to `Task_RequestDocs`, configured with a `bpmn:timerEventDefinition` such as `PT48H` (or a shorter duration like `PT24H` to account for time already consumed in `Task_Receive` and `Task_Check`). Wire a new sequence flow from `ESC_Timer` directly to `Task_Triage`.

Effect: If the claimant does not return the missing documents before the timer fires, the request step is abandoned and the claim is automatically routed to triage — satisfying the 48-hour SLA without introducing any new task. No other flows need to change, and the happy path (complete docs) remains untouched.
