### How the XML maps to the 10 lines

| Line | Element(s) |
|------|-----------|
| 1 | `se` – Start Event |
| 2 | `t1` – Classify Ticket |
| 3 | `gw1` → `f3` → `t2` (Password Reset branch) |
| 4 | `gw1` → `f4` → `t3` (Otherwise branch) |
| 5 | `gw2` splits to `t4` (Resolve) or `t5` (Forward) |
| 6 | `t5` → `f8` → `t4` (specialist work merges into Resolve Issue) |
| 7 | `t4`/`t2` → `t6` (Confirm with Customer) |
| 8 | `gw3` → `f12` → `ee` (customer confirms → end) |
| 9 | `gw3` → `f13` → `t3` (re-work loop back to Initial Diagnosis) |
| 10 | `ee` – End Event |

Three **exclusive gateways** capture the two classification/diagnosis decisions **plus** the confirmation decision that drives the rework loop (line 9). No elements beyond what the description requires were added.
