### Decision points

1. **Eligibility validation**
   - **Triage_Agent_Validate_Eligibility** checks whether the claim is eligible using the *FlightManifest* and *WeatherReport*.

2. **Finance pre-approval**
   - **Finance_Controller_PreApprove** must succeed before any refund payment can begin.
   - The controller signs off using the *EligibilityChecklist*.

3. **High-value threshold**
   - If the refund amount is **greater than €1,000**, the case must go through **Senior_Auditor_Review**.
   - That review must include the *AuditTrail*.
   - If the amount is not greater than €1,000, this review is not required.

4. **Payment readiness**
   - **Process_Refund_Payment** can begin only when:
     - **Finance_Controller_PreApprove** has succeeded;
     - **Senior_Auditor_Review** has been completed for high-value claims; and
     - a signed *BankTransferAuthorization* is present.

### Required documents

| Document | Used for / required when |
|---|---|
| *RefundRequestForm* | Submitted by the customer when starting the claim. |
| *FlightManifest* | Used during eligibility validation. |
| *WeatherReport* | Used during eligibility validation. |
| *EligibilityChecklist* | Used for finance-controller pre-approval. |
| *AuditTrail* | Required for **Senior_Auditor_Review** when the refund exceeds €1,000. |
| *BankTransferAuthorization* | Must be signed and present before refund payment. |
| *RefundConfirmationPDF* | Emailed to the passenger in **Issue_Confirmation_Letter**. |
