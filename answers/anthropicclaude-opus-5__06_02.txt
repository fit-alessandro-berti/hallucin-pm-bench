**Concurrent task sets in Process B — Custom_Furniture_Manufacturing**

1. Generate_CAD_Drawing, Confirm_Materials_Availability  *(Parallel Block P1)*
2. Milling_Panels, Turning_Legs, CNC_Detailing  *(Parallel Block P2)*
3. Sand_Surfaces, Apply_Stain, Install_Hardware  *(Parallel Block P3)*

**Excluded as strictly sequential:** Accept_Custom_Order, Dry_Fit_Assembly, Glue_And_Clamp, Final_Inspection — each is a single-threaded step with a synchronization dependency on the preceding block (P1 → P2 → Dry_Fit_Assembly → Glue_And_Clamp → P3 → Final_Inspection), so no two of them, and none of them with any block member, can overlap.

*Note:* Concurrency exists only **within** each block; the three blocks themselves are gated by join points, so no cross-block pairing (e.g., Turning_Legs with Apply_Stain) is valid.
