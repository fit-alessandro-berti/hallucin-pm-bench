Based only on Contoso’s non-standard process you provided, here are the decision points and required documents:

### **Decision points in Contoso’s process**

1. **Triage_Agent_Validate_Eligibility**  
   - **Decision**: Is the claim eligible based on *FlightManifest* and *WeatherReport*?  
   - If yes → proceeds to Finance_Controller_PreApprove. If no → claim stops.

2. **Finance_Controller_PreApprove**  
   - **Decision**: Does the claim pass sign-off using the *EligibilityChecklist*?  
   - If yes → proceeds to next step. If no → claim stops.

3. **High-value branch check**  
   - **Decision**: Is the refund amount **greater than €1,000**?  
   - If yes → case must go to Senior_Auditor_Review before payment.  
   - If no → can proceed to Process_Refund_Payment after Finance_Controller_PreApprove.

4. **Senior_Auditor_Review** *(only for high-value claims > €1,000)*  
   - **Decision**: Does the case pass auditor review with the *AuditTrail* included?  
   - If yes → proceeds to Process_Refund_Payment. If no → claim stops.

### **All required documents**

1. *RefundRequestForm* — submitted in Customer_Submit_Claim  
2. *FlightManifest* — used in Triage_Agent_Validate_Eligibility  
3. *WeatherReport* — used in Triage_Agent_Validate_Eligibility  
4. *EligibilityChecklist* — used in Finance_Controller_PreApprove  
5. *AuditTrail* — required for Senior_Auditor_Review on high-value claims  
6. *BankTransferAuthorization* — must be signed before Process_Refund_Payment can begin  
7. *RefundConfirmationPDF* — issued to passenger in Issue_Confirmation_Letter

**Constraints noted**: Process_Refund_Payment starts only after Finance_Controller_PreApprove succeeds, plus Senior_Auditor_Review if the claim is > €1,000.
