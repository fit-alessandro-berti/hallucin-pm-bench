• Added tasks  
- `Task_VerifyIdentity` ("Verify Identity"): Inserted at the beginning of the process, preceding the review step.  
- `Task_RequestDocs` ("Request Additional Documents"): New task branching from the decision gateway for missing documentation scenarios.  
- `Task_FinalReject` ("Final Reject"): New task branching from the decision gateway for ineligible scenarios (replaces the previous reject logic under a new ID).  

• Removed tasks  
- `Task_RejectLoan` ("Reject Loan"): The original rejection task has been removed from the model.  

• Gateway-type changes (e.g., Exclusive → Inclusive)  
- `Gateway_Decision`: Changed from an Exclusive Gateway (`exclusiveGateway`) to an Inclusive Gateway (`inclusiveGateway`). This alters the routing behavior, allowing the model to trigger multiple outgoing sequence flows concurrently or conditionally, rather than enforcing a single mutually exclusive path. (Name change from "Eligible?" to "Eligibility Check" treated as cosmetic per instructions.)
