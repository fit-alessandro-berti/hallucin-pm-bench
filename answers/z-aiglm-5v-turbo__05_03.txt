### Changed & Added XML

```xml
    <!-- 1. MODIFIED: Added interrupting Timer Boundary Event to the Service Task -->
    <bpmn:serviceTask id="Activity_CreateAccount" name="Create Account in Core System">
      <bpmn:boundaryEvent id="Timer_SLA_Breach" attachedToRef="Activity_CreateAccount" cancelActivity="true">
        <bpmn:outgoing>Flow_Escalate</bpmn:outgoing>
        <bpmn:timerEventDefinition>
          <bpmn:timeDuration xsi:type="bpmn:tFormalExpression">PT48H</bpmn:timeDuration>
        </bpmn:timerEventDefinition>
      </bpmn:boundaryEvent>
    </bpmn:serviceTask>

    <!-- 2. ADDED: Sequence Flow connecting the Timer to the existing (previously unused) Review Task -->
    <bpmn:sequenceFlow id="Flow_Escalate" sourceRef="Timer_SLA_Breach" targetRef="Activity_AMReview"/>
```

### Justification
By attaching an interrupting **Timer Boundary Event** (set to 48 hours) to the "Create Account" task and routing it to the existing "Account-Manager Review" activity, the model now strictly enforces the SLA deadline by automatically escalating any delayed requests without requiring the creation of new process nodes.
