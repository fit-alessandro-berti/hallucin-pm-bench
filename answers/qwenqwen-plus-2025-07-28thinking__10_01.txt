### RACI Matrix: FinCo Order-to-Cash Workflow  
*(Activities as rows, Roles as columns)*  

| **Activity**             | **Account Manager (AM)** | **Compliance Officer (CO)** | **Finance Clerk (FC)** | **IT Support (IT)** |  
|--------------------------|--------------------------|-----------------------------|------------------------|---------------------|  
| **Receive Application**  | R, A                     |                             |                        |                     |  
| **KYC Review**           | C                        | R, A                        |                        |                     |  
| **Order Vetting**        | R, A                     |                             |                        |                     |  
| **Payment Clearance**    | C                        |                             | R, A                   |                     |  
| **Record Update**        | C                        |                             | C                      | R, A                |  
| **Notify Customer**      | R, A                     |                             | I                      |                     |  

---

### Key:  
- **R** = Responsible (executes the task)  
- **A** = Accountable (owns final approval/decision)  
- **C** = Consulted (provides input before action)  
- **I** = Informed (notified post-completion)  

> **Compliance Note**:  
> - All terminology strictly adheres to FinCoâ€™s policy: **"KYC Review"**, **"Order Vetting"**, and **"Payment Clearance"** used exclusively.  
> - Forbidden terms (e.g., "Customer Due Diligence", "Order Validation", "Payment Processing") are **not referenced** per system constraints.  
> - *Rationale*:  
>   - **KYC Review** (Step 2): CO is Accountable; AM provides supporting documents (Consulted).  
>   - **Order Vetting** (Step 3): AM owns data completeness checks (R/A); no cross-role dependency.  
>   - **Payment Clearance** (Step 4): FC initiates settlement (R/A); AM validates order integrity (Consulted).  
>   - **Record Update** (Step 5): IT executes CRM update (R/A); AM/FC supply change details (Consulted).  
>   - **Notify Customer** (Step 6): AM owns confirmation (R/A); FC confirms payment status (Informed).
