Based on a comparison of the BPMN XML files for Acme Bank's *Loan_Process*, here is the summary of changes:

### • Added tasks
*   **Verify Identity** (`Task_VerifyIdentity`) — Added as a new initial step immediately following the Start Event.
*   **Request Additional Documents** (`Task_RequestDocs`) — Added as a new task handled after the gateway.
*   **Final Reject** (`Task_FinalReject`) — Added to replace the previous rejection step.

### • Removed tasks
*   **Reject Loan** (`Task_RejectLoan`) — Removed (functionally replaced by the newly defined `Task_FinalReject`).

*(Note: The change to `Task_ApproveLoan` from "Approve Loan" to "Auto-Approve Loan" has been ignored as a cosmetic renaming since the underlying XML ID remains unchanged).*

### • Gateway-type changes
*   **Gateway_Decision** (renamed from "Eligible?" to "Eligibility Check") changed its routing logic from an **Exclusive Gateway** (`<exclusiveGateway>`) to an **Inclusive Gateway** (`<inclusiveGateway>`). This allows for parallel evaluation and multi-path activation (e.g., simultaneously approving a loan or requesting documents and rejecting).
