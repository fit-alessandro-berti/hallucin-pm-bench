Add an **interrupting timer boundary event** to `Task_RequestDocs`, with a due date based on submission time: `claimSubmittedAt + PT48H`. Route that timer directly to `Task_Triage`.

So the missing-docs path becomes:

`GW_Docs` → `Task_RequestDocs`
Normal completion: `Task_RequestDocs` → `Task_Triage` via `f6`
Timer expiry: `Timer_48h_RequestDocs` → `Task_Triage`

This adds no new tasks, but prevents cases from waiting indefinitely for missing documents before triage. To make the escalation explicit, name the timer event **“48h SLA Auto-Escalate”** and mark the outgoing timer flow as the escalation path.
