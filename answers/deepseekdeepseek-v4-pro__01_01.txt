**Decision points in Contoso’s refund process**

1. **Eligibility validation**  
   - *Where*: Triage_Agent_Validate_Eligibility  
   - *Condition*: Does the claim pass the checks against the FlightManifest and the WeatherReport?  
   - *Outcome*: If eligible, the claim proceeds; if not, it is rejected (implicit).  

2. **Pre-approval sign-off**  
   - *Where*: Finance_Controller_PreApprove  
   - *Condition*: Does the finance controller sign off using the EligibilityChecklist?  
   - *Outcome*: If succeeded, the claim continues; if not, it is stopped.  

3. **High-value routing**  
   - *Where*: After Finance_Controller_PreApprove  
   - *Condition*: Is the refund amount greater than €1 000?  
   - *Outcome*: If yes → proceed to Senior_Auditor_Review; if no → bypass Senior_Auditor_Review and go directly to Process_Refund_Payment.  

---

**Required documents (artefacts explicitly defined in the process)**  

| Document | Used in activity | Role |
|----------|------------------|------|
| RefundRequestForm | Customer_Submit_Claim | Claim submission |
| FlightManifest | Triage_Agent_Validate_Eligibility | Eligibility check |
| WeatherReport | Triage_Agent_Validate_Eligibility | Eligibility check |
| EligibilityChecklist | Finance_Controller_PreApprove | Pre-approval sign-off |
| AuditTrail | Senior_Auditor_Review | Mandatory for high-value claims |
| BankTransferAuthorization | Process_Refund_Payment | Prerequisite for payment execution |
| RefundConfirmationPDF | Issue_Confirmation_Letter | Final confirmation to passenger |
