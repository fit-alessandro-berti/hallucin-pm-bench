### AtlasRemit Cross-Border Payment Process RACI Matrix

Below is the RACI matrix for the AtlasRemit cross-border payment process. It defines the roles and responsibilities for each task, strictly utilizing AtlasRemit’s mandated vocabulary.

| Task / Activity | Transaction Analyst | Regulatory Liaison | IT Support |
| :--- | :---: | :---: | :---: |
| **1. Receive and log transfer instruction** <br>Receive customer's transfer instruction and log it in the system. | **A/R** | | |
| **2. Sanctions screening** <br>Screen the logged payment against the sanctions list. | **A/R** | **C** | |
| **3. Perform KYC Review** <br>Perform the **KYC Review** on both the sender and the beneficiary. | | **A/R** | |
| **4. High-risk transaction approval** <br>Review and approve the transaction if flagged as high-risk. | | **A/R** | |
| **5. Payment release** <br>Release the approved payment. | **A/R** | **I** | |
| **6. SWIFT transmission** <br>Ensure the SWIFT message is successfully sent. | | | **A/R** |
| **7. Case archiving & notification** <br>Archive the final case file and trigger notification of completion. | | **I** | **A/R** |

***

### Key to RACI Roles:
*   **R – Responsible:** The role that performs the activity/does the work.
*   **A – Accountable:** The role with final approval and ownership of the step (only one 'A' assigned per task).
*   **C – Consulted:** The role that provides input, feedback, or guidance.
*   **I – Informed:** The role kept up-to-date on progress or outcomes.
