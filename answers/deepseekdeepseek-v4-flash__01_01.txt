## Decision Points in Contoso Airlines' Refund Process

1. **Triage_Agent_Validate_Eligibility** – The triage agent must decide whether the claim is eligible based on the *FlightManifest* and *WeatherReport*. (No explicit criteria are provided; the agent exercises judgement.)
2. **Finance_Controller_PreApprove** – The finance controller must decide whether to sign off the claim using the *EligibilityChecklist*.
3. **High-value branch (implicit gateway)** – If the refund amount **> €1,000**, the process must include **Senior_Auditor_Review** before proceeding to payment.
4. **Process_Refund_Payment readiness** – The system (or finance team) must verify that **both** *Finance_Controller_PreApprove* has succeeded **and** (for high-value claims) *Senior_Auditor_Review* has been completed before executing payment.

*Note:* The legacy activity “CustomerService_Agent_Decide” does **not** exist in Contoso’s process.

## Required Documents

| Document | Used In |
|----------|---------|
| *RefundRequestForm* | Customer_Submit_Claim |
| *FlightManifest* | Triage_Agent_Validate_Eligibility |
| *WeatherReport* | Triage_Agent_Validate_Eligibility |
| *EligibilityChecklist* | Finance_Controller_PreApprove |
| *AuditTrail* | Senior_Auditor_Review (only for high-value claims) |
| *BankTransferAuthorization* (signed) | Process_Refund_Payment |
| *RefundConfirmationPDF* | Issue_Confirmation_Letter |
