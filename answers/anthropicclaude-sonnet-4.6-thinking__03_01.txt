### Key modelling decisions

| Line(s) | BPMN element | Rationale |
|---------|-------------|-----------|
| 2 | `t1 – Classify Ticket` (plain Task) | "automatically classified by the system" – no explicit service-task dialect required |
| 3 | `gw1` exclusive gateway → `t2` / `t3` | First decision point: Password Reset vs. otherwise |
| 5 | `gw2` exclusive gateway → `t5 (Resolve Issue)` / `t4 (Forward to Specialist)` | Second decision point after diagnosis |
| 6 | `t4 → t5` | Specialist path converges back into the single **Resolve Issue** task |
| 7 | `t2 → t6` | Password-reset path also reaches **Confirm with Customer** (only exit the process has) |
| 8–9 | `gw3` exclusive gateway → `e2` / loop back to `t3` | Third gateway (customer confirmed vs. issue persists re-work loop) |
