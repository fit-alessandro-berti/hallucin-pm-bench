### Key Features:
1. **Flow Elements**: Only includes explicitly mentioned tasks, gateways, and events.
2. **Gateways**:
   - Exclusive gateway (`wrc_gateway_completeness`) for claim completeness check.
   - Exclusive gateway (`wrc_gateway_eligibility`) for warranty decision.
   - Parallel gateway pair (`wrc_parallel_split` and `wrc_parallel_join`) for concurrent tasks.
3. **End Events**: Two end events as required (`wrc_end_rejection` and `wrc_end_success`).
4. **Naming**: All IDs prefixed with `wrc_`, and task names match the description verbatim.
5. **Conditions**: Placeholder expressions (e.g., `#{claimIncomplete}`) for routing logic.
