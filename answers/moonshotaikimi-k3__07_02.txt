**Added tasks**
- **Verify Identity** (`Task_VerifyIdentity`) — new task inserted between Start and Review Application.
- **Request Additional Documents** (`Task_RequestDocs`) — new task on a third outgoing branch ("missing docs") of the decision gateway.
- **Final Reject** (`Task_FinalReject`) — new task handling the "ineligible" outcome; it functionally takes over the rejection role, but since it has a different ID from the v1 rejection task, it counts as an addition rather than a rename.

**Removed tasks**
- **Reject Loan** (`Task_RejectLoan`) — removed; its rejection branch is now served by the new Final Reject task.

**Gateway-type changes**
- `Gateway_Decision`: **exclusiveGateway → inclusiveGateway**. (Its outgoing flows consequently changed from two mutually exclusive paths to three: eligible / missing docs / ineligible.)

*Note: the renamings "Approve Loan" → "Auto-Approve Loan" (same ID) and the gateway label "Eligible?" → "Eligibility Check" were treated as cosmetic and ignored.*
