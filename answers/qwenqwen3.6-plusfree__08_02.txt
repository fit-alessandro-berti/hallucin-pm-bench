**Q2.** Sequence: `T1, T2, T7, T4, T5`. 
Explanation: T1 produces exactly one token in P2. T3 and T5 structurally conflict over P2, so only one can fire per run. By routing the parallel branch via `T2 → T7 → T4`, we generate a token in P5 while leaving P2 untouched. This enables T5, which fires and consumes the sole P2 token. Because T5 executes and exhausts P2, it legally fires before T3 (which never becomes enabled again). The single-token constraint on P2 inherently enforces this ordering.
