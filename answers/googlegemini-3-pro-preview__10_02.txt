Here is the RACI matrix for the **AtlasRemit** payment workflow, adhering to the mandated vocabulary.

### **Process: Cross-Border Payment Execution**

| Task Description | Transaction Analyst | Regulatory Liaison | IT Support |
| :--- | :---: | :---: | :---: |
| **1. Log Transfer Instruction** <br>Receive instruction and enter data into system. | **R / A** | I | - |
| **2. Sanctions Screening** <br>Screen payment against sanctions lists. | **R** | **A** | - |
| **3. Perform KYC Review** <br>Conduct review on sender and beneficiary. | I | **R / A** | - |
| **4. High-Risk Approval** <br>Review and approve flagged transactions. | C | **R / A** | - |
| **5. Release Payment** <br>Execute payment release in the system. | **R / A** | I | C |
| **6. Transmit SWIFT Message** <br>Ensure technical transmission of the message. | I | - | **R / A** |
| **7. Archive Case File** <br>Finalize and store transaction records. | - | **I** | **R / A** |

---

**RACI Key:**
*   **R (Responsible):** The role doing the work to complete the task.
*   **A (Accountable):** The role answerable for the correct and thorough completion of the task.
*   **C (Consulted):** The role whose opinions are sought; two-way communication.
*   **I (Informed):** The role that is kept up-to-date on progress; one-way communication.
