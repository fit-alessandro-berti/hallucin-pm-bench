**Add an interrupting timer boundary event (e.g., `Timer_SLA48`, duration ≈ PT44H) to `Task_RequestDocs`, with an outgoing sequence flow to `Task_Triage`.**

`Task_RequestDocs` is the only step where the process can stall indefinitely (waiting on the claimant), risking the 48-hour SLA. With the timer attached, if missing documents don't arrive in time, the wait is interrupted and the claim is automatically routed to `Task_Triage` — effectively auto-escalating it without adding any new task. The timer duration is set slightly below 48 hours to allow for the upstream processing time of `Task_Receive` and `Task_Check`.

The happy path (`f6`) remains unchanged: if docs arrive in time, the boundary timer is simply cancelled and triage proceeds normally.
